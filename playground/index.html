🔄

Idempotent API Requests

The frontend generates a unique Idempotency Key for every checkout attempt. This ensures that if a network timeout occurs and the client retries, the backend won't double-charge or reserve stock twice.

🔒

Deadlock Prevention (Pessimistic Locking)

When checking out multiple items, the backend sorts all requested products alphabetically by ID before acquiring Row-Level Postgres Locks. This strict ordering prevents database deadlocks when concurrent users checkout intersecting carts.
